# Aptor two-minute demo script

Use the hosted in-platform flow and the already completed real Preprod scenario.
Do not show file import/export, repeat transactions, or substitute transaction
IDs.

## 0:00–0:20 — The confidential-work problem

“A professional's strongest work is often confidential. Aptor lets that work
speak without revealing the client, repository, project, or exact private
values.” Show the Professional workspace and privacy boundary.

## 0:20–0:40 — Invite the previous client

Create an Issuer invitation from the Professional profile. Open the public
invite URL in the isolated Issuer profile and redeem it.

## 0:40–1:00 — Encrypted credential delivery

Enter one bounded demo credential. Sign it locally as the Issuer, encrypt it to
the Professional, and send it through Aptor. Switch to the Professional inbox,
decrypt, validate, and accept it. State that the server routed ciphertext and
metadata, not credential plaintext.

## 1:00–1:20 — Register a requirement

In the isolated Verifier profile, select the Professional and accepted Issuer,
define the skill/duration/production/rating threshold, connect 1AM, and show the
already finalized registration receipt. Send the registered request through
Aptor.

## 1:20–1:45 — Prove privately

Open the request in the Professional inbox. Show the local compatible-credential
match and disclosure review. Connect 1AM and show the already completed real
proof result; do not risk a duplicate/replay transaction during recording.

## 1:45–2:00 — Verify the receipt

Return to the Verifier. Show **Request fulfilled**, the public transaction and
contract receipt, and the absence of credential files or exact work values.

End card: **Aptor — Prove the work. Protect the details.**
